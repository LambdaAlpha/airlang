# 上下文

## read

类型：function

上下文访问：不变

调用模式：`{symbol : form.}`

输出符号 i 所绑定的值。

## move

类型：function

上下文访问：可变

调用模式：`{symbol : form.}`

移除并输出符号 i 所绑定的值。

## =

类型：function

上下文访问：可变

调用模式：`{pair : form. : .}`

模式匹配与赋值。

### 绑定形式

若 i 为 `bind : value`，`bind` 的一般形式为 `pattern . options`。

`pattern` 是一个模式，用于和输入进行模式匹配。

`options` 是一些选项，用于指定不变性等。

若使用当前的默认选项，则可以省略 `options`，`bind` 表示为 `pattern`。

### 选项

选项是一个映射。

- 键 `invariant` 表示不变性
  - 值 none 表示无不变性
  - 值 final 表示不可赋值
  - 值 constant 表示不可变
  - 默认值为 none

默认选项是 `{}`。

一些简单的选项具有简写形式，如

- `final` 表示 `{invariant : final}`
- `constant` 表示 `{invariant : constant}`

### 匹配绑定

匹配绑定 `bind` 与 `value` 的过程：

- 若`bind` 中存在 `options`，则设置为当前默认选项。当前默认选项在递归时有效。

- 若 `pattern` 为符号 `a`，则匹配任意值 `v`，按照当前默认选项将 `v` 绑定到 `a`，输出 `c`，`c` 为 `a` 当前所绑定的值或单元
- 若 `pattern` 为 `b1 : b2`（`b1 ; b2`，`b1 ! b2`，`b1 ? b2`），则匹配 `v1 : v2`（`v1 ; v2`，`v1 ! v2`，`v1 ? v2`），递归
  - 将 `v1` 与 `b1` 匹配绑定，得到 `cv1`
  - 将 `v2` 与 `b2` 匹配绑定，得到 `cv2`
  - 输出 `cv1 : cv2`（`cv1 ; cv2`，`cv1 ! cv2`，`cv1 ? cv2`）
- 若 `pattern` 为 `[b1, b2, ..., bn]`，则匹配列表 `l`，递归
  - 将 `l` 的第一个值或单元与 `b1` 匹配绑定，得到 `cv1`
  - 将 `l` 的第二个值或单元与 `b2` 匹配绑定，得到 `cv2`
  - 将 `l` 的第 `n` 个值或单元与 `bn` 匹配绑定，得到 `cvn`
  - 输出 `[cv1, cv2, ., cvn-1, cvn]`
- 若 `pattern` 为 `{k1 : b1, k2 : b2}`，则匹配映射 `m`，递归
  - 将 `m` 中 `k1` 所绑定的值或者单元与 `b1` 匹配绑定，得到 `cv1`
  - 将 `m` 中 `k2` 所绑定的值或者单元与 `b2` 匹配绑定，得到 `cv2`
  - 输出 `{k1 : cv1, k2 : cv2}`

## set_invariant

类型：function

上下文访问：可变

调用模式：`{pair : {symbol : form.} : {symbol : form.}}`

若 i 未 `name : invariant`，且 `name` 已绑定值，则设置 `name` 的不可变性为 `invariant`。若当前上下文不可回退，且当前的不可变性大于 `invariant`，则此操作会失败。

## invariant

类型：function

上下文访问：不变

调用模式：`{symbol : form.}`

询问模式：`{symbol : form.}`

若符号 i 已绑定值，则输出其不可变性。

- none
- final
- constant

## fallback

类型：function

上下文访问：不变

输出当前上下文是否可回退

## is_null

类型：function

上下文访问：不变

调用模式：`{symbol : form.}`

输出符号 i 是否未绑定值。

## access

类型：function

上下文访问：不变

询问模式：`{symbol : form.}`

输出当前上下文的可访问性。

- free 表示上下文无关
- constant 表示上下文不变
- mutable 表示上下文可变

## solver

类型：function

上下文访问：不变

输出当前上下文的求解器。

## set_solver

类型：function

上下文访问：可变

将上下文无关函数 i 设置为当前上下文的求解器。若 i 为单元，则清除当前上下文的求解器。

## |

类型：function

上下文访问：可变

调用模式：`{pair : {symbol : form.} : form*}`

可缓存性：否

在目标上下文中求值。

若 i 为 `ctx : v`：

`ctx` 表示目标上下文。

- 单元表示无上下文
- 符号 `a` 表示键 `a` 所绑定的上下文

## |:

类型：function

上下文访问：无关

调用模式：`{pair : . : form*}`

可缓存性：否

若 i 为 `ctx : i`，则在 `ctx` 中求值 `i`，得到 `o`，输出 `ctx : o`。

## context

类型：function

上下文访问：无关

调用模式：`{map : {variables : {map : {} : {symbol : form} : .}} : {symbol : form.}}`

构建一个上下文。

- variables 的键值对为 `k : bind`，`bind` 的一般形式为 `value . options`。
  - `options` 是一些选项，用于指定不变性等。
  - `bind` 也可以省略 `options`：`value`，表示使用默认的选项。
  - 按照 `options` 选项将 `value` 绑定到键 `k` 上。
- fallback：可回退性
- solver：求解器

### 选项

（大致同 `=`，多了 `static` 部分）

选项是一个映射

- 键 `invariant` 表示不变性
  - 值 none 表示无不变性
  - 值 final 表示不可赋值
  - 值 constant 表示不可变
  - 默认值为 none
- 键 `static` 表示静态性，值为比特类型

默认选项是 `{}`。

一些简单的选项具有简写形式，如

- `final` 表示 `{invariant : final}`
- `constant` 表示 `{invariant : constant}`
- `static` 表示 `{static : true}`

## context.represent

表示一个上下文。
